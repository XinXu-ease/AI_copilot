## 🚀 快速开始

### 方式 1：使用启动脚本（推荐）

```bash
# 安装依赖
pip install -r requirements.txt

# 运行启动脚本
python run.py
```

这将自动：
- 启动 Flask API 后端（localhost:5000）
- 启动前端 HTTP 服务（localhost:8000）
- 打开浏览器访问 `http://localhost:8000/index.html`

### 方式 2：手动启动

**终端 1 - 启动 API 后端：**
```bash
python app_api.py
```

**终端 2 - 启动前端服务：**
```bash
python -m http.server 8000
```

**浏览器：**
访问 `http://localhost:8000/index.html`

## 🔌 API 接口

### 1. 启动项目
```
POST /api/project/start
Content-Type: application/json

{
  "idea": "我想做一个AI驱动的笔记应用"
}

Response:
{
  "success": true,
  "project_id": "uuid",
  "state": { ... 项目状态 ... }
}
```

### 2. 获取项目状态
```
GET /api/project/{project_id}

Response:
{
  "success": true,
  "state": { ... 当前项目状态 ... }
}
```

### 3. 提交澄清问题答案
```
POST /api/project/{project_id}/clarification
Content-Type: application/json

{
  "answers": [
    "答案1",
    "答案2",
    "答案3",
    "答案4",
    "答案5"
  ]
}

Response:
{
  "success": true,
  "state": { ... 更新后的项目状态 ... }
}
```

## 📊 工作流程

```
1. 用户输入想法
       ↓
2. API 调用启动项目
       ↓
3. PM Agent 生成 Brief
       ↓
4. 检查是否需要澄清
       ├─→ 需要：显示澄清问题 → 用户填写 → 回到 3
       └─→ 不需要：继续
       ↓
5. Research Agent 第一轮研究
       ↓
6. Research Agent 第二轮深度研究
       ↓
7. UX Agent 生成设计方案
       ↓
8. Developer Agent 生成开发计划
       ↓
9. 显示完整结果
```

1. **状态存储** - 目前使用内存存储，服务器重启会丢失数据
   - 生产环境建议改为数据库存储

2. **CORS 设置** - 已配置允许跨域请求

3. **性能优化** - 第一次调用可能需要 1-3 分钟（LLM 处理时间）

## ❓ 常见问题

**Q: 为什么改成 HTML？**
A: 提供更灵活的 UI 定制，避免 Streamlit 的局限

**Q: 会不会变慢？**
A: 性能基本相同，实际延迟主要来自 LLM 调用

**Q: 能恢复 Streamlit 版本吗？**
A: 可以，运行 `streamlit run app.py`

**Q: 数据会保存吗？**
A: 目前只在内存中，重启后丢失。可修改为数据库存储
